Контекст:
У нас есть адрес базы данных внешней системы, содержащей адреса в разной иерархии:
URl Базы данных: http://address.ru/
В базе данных лежат части адреса в российской локализации в следующей иерархии:
1. region - регион, округ, край и т.д.
2. city - город, поселок, село, деревня, станица и т.д.
3. street_address - улица, аллея, бульвар и т.д.
4. postcode - индекс

Есть требования к возвращаемым адресам:
1. region:
1.1 должен содержать одно из следующих значений:
"респ.", "АО", "обл.", "край"
2. city:
2.1 должен содержать одно из следующих значений:
"п.", "с.", "клх", "д.", "к.", "ст.", "г."
3. street_address:
3.1 должен содержать одно из следующих значений:
"пер.", "ш.", "пр.", "алл.", "наб.", "ул.", "бул."
3.2 Также в street_address в каждом ответе должно быть значение "д." после типов улиц из пункта 3.1
4. postcode должен содержать 6 цифр, не должно вернуться 000000

1. При отправке одной из 4-х частей адреса на сервер нам возвращается предлагаемый список адресов full_address,
содержащих в себе region, city, street_address, postcode.

2. Условия:
2.1 В предлагаемом списке адресов в обязательном порядке имеется отправляемая нами часть адреса.
2.2.1 При отправке region нам возвращается 30 адресов
2.2.2 При отправке city нам возвращается 20 адресов
2.2.3 При отправке city нам возвращается 10 адресов
2.2.4 При отправке postcode нам возвращается 5 адресов
2.3 Если адрес валидный - 200, если нет - 404;

3. Реализовать запись валидных адресов в файл.
4. Файл с адресами сохранять в репозитории.
